TEMA 1: Principios de Seguridad y Sanitizaciﾃｳn XSS en LocalStorage
------------------------------------------------------------------------

**Prompt sugerido para la IA:**
"Estoy desarrollando una aplicaciﾃｳn SPA en React donde los usuarios pueden crear y editar productos a travﾃｩs de formularios. Estos productos se guardan en el LocalStorage del navegador.
ﾂｿQuﾃｩ medidas de seguridad debo implementar para evitar ataques de Cross-Site Scripting (XSS) al almacenar y posteriormente renderizar strings ingresados por el usuario?
Proporciﾃｳname una funciﾃｳn helper en JavaScript para sanitizar las entradas de texto antes de guardarlas y explica cﾃｳmo aplicarla en los inputs."

**Resumen de la recomendaciﾃｳn de seguridad implementada:**
1. Sanitizaciﾃｳn de caracteres peligrosos antes del guardado: se deben escapar caracteres de control HTML como `<`, `>`, `&`, `"`, `'`, `/` para evitar que el navegador interprete texto del usuario como etiquetas ejecutables.
2. Uso de formularios controlados en React: React por defecto escapa las variables que se renderizan dentro de las llaves `{}` (JSX), a menos que se use `dangerouslySetInnerHTML`. Sin embargo, es buena prﾃ｡ctica sanitizar antes de guardar en base de datos local (localStorage) para garantizar la integridad de los datos estructurados.

**Funciﾃｳn de escape implementada en el proyecto:**
const sanitizeInput = (str) => {
  if (typeof str !== 'string') return ''
  return str
    .replace(/&/g, '&amp;')
    .replace(/</g, '&lt;')
    .replace(/>/g, '&gt;')
    .replace(/"/g, '&quot;')
    .replace(/'/g, '&#x27;')
    .replace(/\//g, '&#x2F;')
}
